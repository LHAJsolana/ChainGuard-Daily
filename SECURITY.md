# Security policy

## Reporting

Do not open a public issue for a suspected vulnerability. Contact the project owner privately and include the affected route, reproduction steps, impact, and any relevant transaction signature. Never include seed phrases, private keys, or other credentials.

## Product boundary

ChainGuard Daily reads public Solana data. It never needs wallet connection, signing approval, private keys, or seed phrases. Any site or person claiming otherwise is not following this repository's design.

Reports are heuristic technical signals, not a formal audit, guarantee of safety, or financial advice.

## Supported release

Until a stable release policy is published, only the latest deployment from the default branch receives security updates.
